非阻塞同步实现

发表于 2016-08-25   |   分类于 Java

概念

  • 阻塞同步：即互斥同步，包括synchronized,Lock 。悲观锁，总是认为数据会被更改，无论共享的数据是否会出现竞争，都会进行加锁，用户态和内核态转换，维护锁计数器和检查是否有被阻塞的线程需要唤醒等操作，耗费性能。
  • 非阻塞同步：采用乐观锁，当数据产生了冲突才采取补偿措施（最常见的补偿措施就是不断的重试，直到成功为止），这种乐观的并发策略不需要把线程挂起，极大提升了性能

实现非阻塞同步条件

需要硬件指令集来支持如CAS指令（比较并交换）
CAS指令需要三个操作数：V(内存地址)，A(预期值)，B（新值）
当执行CAS指令的时候，只有V符合预期的A值时，才会将V的值更新为B，否则就不更新，并且无论是否更新成功，都会返回V的旧值

具体实现

这里分析要J.U.C下的AtomicInteger类，大部分方法都使用了Unsafe类提供的CAS操作

缺陷

ABA问题：如果一个变量为A值，如果他被改为B值后又被改为A值，此时，CAS操作就认为它从来没有改变过，然而实际上他改变过了。
一般情况下，ABA问题不会影响并发程序的正确性，倘若要解决ABA问题，可以改用互斥同步。
