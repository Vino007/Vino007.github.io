发表于 2017-09-29

Thrift是什么

Thrift是一个跨语言的服务部署框架，最初由Facebook于2007年开发，2008年进入Apache开源项目。Thrift通过IDL（Interface Definition Language，接口定义语言）来定义RPC（Remote Procedure Call，远程过程调用）的接口和数据类型，然后通过thrift编译器生成不同语言的代码，并由生成的代码负责RPC协议层和传输层的实现。

IDL

接口定义语言，用于跨语言的RPC服务接口的定义
thrift支持的类型

基本类型

bool，布尔型，1个字节；
byte，有符号单字节；
i16，有符号16位整型；
i32，有符号32位整型；
i64，有符号64位整型；
double，64位浮点数；
string，字符串；
binary，字节数组；
注意：thrift不支持无符号整型。

容器

map，字典；
list，列表；
set，集合；
注意：容器中的元素类型可以是除了service 以外的任何合法thrift类型（包括结构体和异常）。
与JAVA的类型的对应关系
bool: boolean
binary: byte[]
byte: byte
i16: short
i32: int
i64: long
double: double
string: String
list: List
set: Set
map: Map

结构体

结构体中，每个字段包含一个整数ID，数据类型、字段名，和一个可选的默认值。

###tips：

  • 不能继承
  • optional比required更适用

    异常

    异常在语法和功能上类似于结构体，只不过异常使用关键字exception而不是struct关键字声明。但它在语义上不同于结构体，当定义一个RPC服务时，开发者可能需要声明一个远程方法抛出一个异常。

    Service

    定义接口
    参数和返回类型必须是thrift基本类型或者struct
    允许继承，关键词 extend

typedefs

Constants as public static final members. Constants of any of the primitive types are supported.

IDL的替代方案——Swift

原因：IDL自动生成的代码太冗长复杂

Thrift的Java注解实现

注解说明
service <-> @ThriftService
struct <-> @ThriftStruct
struct 属性 <-> @ThriftField
enum <-> @ThriftEnumValue
method<->ThriftMethod
constuctor↔ThriftConstructor
field ↔ ThriftField
枚举类型可以直接传输

开发过程中异常处理

java.lang.IllegalArgumentException: Type can not be coerced to a Thrift type:

检查@ThriftStruct注解有没有标记上
swift.codec.metadata.MetadataErrorException: Error: Metadata extraction encountered 1 errors and 0 warnings
原因有三点：
1.getter、setter方法命名没有与属性名对应
2.Thrift不支持这个类型比如DateTime、InputStream

  1. getter 、setter、 构造器上缺少ThriftField，ThriftCOnstructor注解
    java.lang.IllegalArgumentException: Multiple entries with same key:
    原因在于thrift不支持重载，因此同名方法即使参数不同也要使用 @ThriftMethod(“name”)标识method名
